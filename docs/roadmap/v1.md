# v1 Specification

## Purpose

Define **version 1.0** — the first complete product release with full city navigation, all districts, and simulation.

---

## v1 Goal

> A user can navigate from Earth to any district, building, room, and agent across a living city of 500 agents with real-time simulation.

---

## Scope Beyond MVP

| Feature                   | Detail                                   |
| ------------------------- | ---------------------------------------- |
| Earth view                | Rotating globe with megacity beacon      |
| Solar System view         | Orbiting planets; Earth entry point      |
| Orbital Ring              | Segment navigation with status           |
| All 5 districts           | Full 3D detail with unique themes        |
| 40 buildings per district | 200 total buildings                      |
| 500 agents                | Distributed across all districts         |
| Scale transitions         | Animated camera flights (all city-scale) |
| Simulation tick           | 60-second world state updates            |
| Search                    | Cross-entity search with quick-jump      |
| Mobile UI                 | Bottom sheet navigation                  |
| Authentication            | JWT-based sessions                       |
| District transit          | Monorail lines between districts         |
| Building interiors        | All building types have 1+ rooms         |

---

## New Technical Components

| Component                 | Technology                       |
| ------------------------- | -------------------------------- |
| ScaleTransitionController | Bezier camera paths              |
| Solar System scene        | Simplified orbital paths         |
| Earth scene               | Atmosphere shader, cloud layer   |
| Orbital Ring scene        | Instanced segments               |
| Simulation engine         | Bull cron, 60s tick              |
| Search service            | PostgreSQL full-text + vector    |
| Auth module               | JWT + refresh tokens             |
| District procgen          | Parameterized building placement |
| Mobile layout             | Responsive bottom sheet          |

## v1 Success Criteria

| Criterion                       | Target                    |
| ------------------------------- | ------------------------- |
| Earth to agent navigation       | < 30 s (with transitions) |
| All 5 districts distinguishable | < 2 s recognition         |
| 500 agents, 200 buildings       | ≥ 30 FPS desktop          |
| Simulation tick                 | < 5 s processing time     |
| Search results                  | < 500 ms                  |
| Concurrent users                | 1,000                     |
| Uptime                          | 99.5%                     |

## Tradeoffs

| Decision                    | Rationale                    | Cost                             |
| --------------------------- | ---------------------------- | -------------------------------- |
| No galaxy view in v1        | Galaxy deferred to v2        | Incomplete cosmic stack until v2 |
| Auth added but not required | Governor role optional       | Complexity without forcing login |
| 500 not 5,000 agents        | AI cost and rendering budget | City feels less populated        |

---

## Implementation Guidance

v1 builds directly on MVP codebase. No rewrites — extend scenes, add districts, scale agent count.
