# Future Vision

## Purpose

**Aspirational target** — the civilization we are building toward. Use for roadmap planning, world-bible consistency, and design direction.

> **Do not implement** features listed here unless they are promoted to [`../current-state/`](../current-state/) or explicitly scoped in [`../memory/active-work.md`](../memory/active-work.md).

---

## Why This Is Separate

Mixing current reality with future dreams confuses AI agents — they generate Galaxy scenes, 10,000-agent swarms, and voice pipelines when the MVP is a single district and 50 agents.

| Doc                                      | Role                                                  |
| ---------------------------------------- | ----------------------------------------------------- |
| [`../current-state/`](../current-state/) | **Build this**                                        |
| **This folder**                          | **Dream this** (phased delivery)                      |
| [`../world-bible/`](../world-bible/)     | **Describe the full fictional world** (lore + design) |
| [`../roadmap/`](../roadmap/)             | **Schedule** when vision phases ship                  |

## Rules for Cursor

1. **Do not** scaffold Galaxy, Earth, governance, simulation, or voice from this folder alone.
2. Feature specs under `feature-specs/` may describe future phases — check **Status** and [`../current-state/capabilities.md`](../current-state/capabilities.md) before coding.
3. Promoting a feature to current requires: update `current-state/`, trim `future-vision/`, update roadmap milestone, and optionally an ADR.
